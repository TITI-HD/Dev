Comment obtenir son nom d'utilisateur et mot de passe d'application

Pour vous connecter à l'API WordPress, vous avez besoin d'un nom d'utilisateur et d'un mot de passe d'application (App Password).

Pour WordPress.com

1

Connectez-vous à votre compte WordPress.com et allez dans votre Tableau de bord.

2

Allez dans Mon profil puis dans l'onglet Sécurité.

3

Descendez jusqu'à la section Mots de passe d'application.

4

Donnez un nom à votre application (ex: "Sauvegarde API") et cliquez sur Ajouter.

5

Copiez le mot de passe d'application qui s'affiche. Notez-le dans un endroit sûr, car vous ne pourrez plus le voir ultérieurement.

Attention : Le mot de passe d'application est différent de votre mot de passe principal. Il est conçu spécifiquement pour les applications tierces.

Pour WordPress auto-hébergé

1

Connectez-vous à l'administration de votre site WordPress.

2

Allez dans Utilisateurs → Votre profil.

3

Descendez jusqu'à la section Mots de passe d'application.

4

Donnez un nom à votre application et cliquez sur Ajouter un nouveau mot de passe d'application.

5

Copiez le mot de passe qui s'affiche. Notez-le dans un endroit sûr.

Votre nom d'utilisateur est simplement votre identifiant WordPress habituel. Utilisez-le avec le mot de passe d'application que vous venez de générer.

À quoi sert le test d'authentification dans votre projet

Le test d'authentification WordPress est une étape cruciale pour vérifier que votre application peut communiquer correctement avec l'API WordPress.

Pourquoi c'est important

🔒 Vérification de la sécurité

Le test permet de s'assurer que vos identifiants sont corrects et que vous avez les permissions nécessaires pour accéder aux données.

🛠️ Dépannage des problèmes de connexion

Si votre application ne parvient pas à récupérer les données, le test d'authentification vous aide à identifier la source du problème (identifiants incorrects, URL erronée, etc.).

📊 Validation des endpoints API

Vous pouvez tester différents endpoints (posts, pages, utilisateurs) pour vous assurer qu'ils répondent correctement.

⚙️ Configuration initiale

Avant de développer votre application, le test vous permet de vérifier que tout est correctement configuré.

Cas d'utilisation dans votre projet

1

Sauvegarde de données : Vérifiez que vous pouvez accéder aux posts et pages pour les sauvegarder.

2

Synchronisation : Testez la connexion avant de configurer une synchronisation régulière avec une application externe.

3

Migration : Assurez-vous que l'API est accessible avant de tenter une migration de contenu.

4

Développement : Utilisez le test pendant le développement pour déboguer les problèmes d'accès à l'API.

En résumé, le test d'authentification vous évite des heures de débogage en vérifiant d'abord que la connexion de base fonctionne correctement.
